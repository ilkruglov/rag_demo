# Universal RAG Demo Engine

Универсальный движок Retrieval-Augmented Generation (RAG) для демонстраций. Основа — модели Groq (Qwen, LLaMA и др.), локальный Qdrant и веб-интерфейс Streamlit. Пользователь загружает документы, удаляет их из базы при необходимости и получает ответы с источниками.

## Возможности

- **Гибридный ретривал.** Используется векторный индекс Qdrant и keyword-индекс. QueryFusionRetriever объединяет их (reciprocal rank fusion), что помогает находить релевантность даже для терминов без хороших эмбеддингов.
- **Управление документами.** Через API и Streamlit можно добавлять и удалять документы с автоматической переиндексацией.
- **Настройки без кода.** Из UI доступны системный промпт/пресеты, параметры LLM (`temperature`, `top_p`, `max_tokens`), `Top-K`, стоп-последовательности, выбор реранкера (keyword overlap или cross-encoder) и его `Top-N`.
- **Богатые метаданные.** Ответы содержат источники: название документа, страницу, раздел, категорию, выдержку и score. При нехватке данных бот корректно сообщает, что информации нет.
- **Автоматическое переиндексирование.** После загрузки или удаления документов запускается ingest, который нормализует текст, делит его на чанки и обновляет оба индекса.

## Правила ответа

Системный промпт закреплён в `app/config.py` и по умолчанию универсален:

- использовать только факты из контекста; при их отсутствии честно сообщать об этом и предлагать, что уточнить;
- не раскрывать внутренние рассуждения и не выводить `<think>`/`analysis`;
- форматировать ответ как `**Ответ:** …` и `**Источники:** …`.

Промпт и параметры можно менять из UI. Есть пресеты для универсальных сценариев. Сохранённые значения попадают в `config/runtime_settings.json`, кеш RAG автоматически сбрасывается.

## Стек

- **Backend:** FastAPI + uvicorn.
- **RAG:** LlamaIndex (VectorStoreIndex + SimpleKeywordTableIndex, QueryFusionRetriever, кастомный KeywordOverlapReranker / SentenceTransformerRerank).
- **Хранилище:** локальный Qdrant (векторное), JSON-хранилище LlamaIndex (keyword).
- **LLM:** Groq через `langchain-groq`.
- **Streamlit UI:** вкладки «Чат и документы» и «Настройки».
- **Подготовка документов:** `unstructured`, `docx2txt`, `pypdf`, SentenceSplitter.
- **Инфраструктура:** Pydantic Settings, python-multipart, pytest.

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

Скопируйте пример окружения и задайте ключи:

```bash
cp .env.example .env
```

Минимально нужен `GROQ_API_KEY` (https://console.groq.com). Остальные переменные (`QDRANT_PATH`, `STORAGE_DIR`, `ingest_chunk_size`, `ingest_chunk_overlap` и т.д.) можно задать в `.env` или через интерфейс.

### Python 3.13

- Перед установкой зависимостей обновите системный `pip`, `setuptools` и `wheel`, иначе многие колёса для 3.13 не подтянутся.
- Стек эмбеддингов LlamaIndex опирается на `sentence-transformers` и `torch`. Используйте официальные сборки PyTorch, совместимые с Python 3.13 (на момент релиза они распространяются через отдельные колёса/репозитории).
- Экстра `uvicorn[standard]` не ставится на 3.13 из-за `uvloop`. Если нужен hot reload, установите `watchfiles` вручную либо работайте на базовом `uvicorn`.
- Если `sentence-transformers` недоступен, сервис автоматически откатится к легкому keyword-реранкеру, но для построения эмбеддингов ingestion всё равно потребуется совместимый стек Hugging Face.

### Работа без интернета

По умолчанию приложение может скачивать модели торч/`sentence-transformers` с Hugging Face во время первой индексации. Если планируется запуск в полностью изолированной среде, задайте переменную окружения `FORCE_OFFLINE_MODE=1` (или поставьте аналогичное значение в настройках) — тогда все обращения к Hugging Face останутся в офлайн-режиме, а необходимые файлы нужно предварительно положить в локальный кэш.

## Переиндексация

```bash
source .venv/bin/activate
python -m scripts.ingest
```

Скрипт читает документы из `data/raw`, нормализует текст, вычисляет эмбеддинги, создаёт/обновляет индексы `nrd_vector_index` и `nrd_keyword_index`, а затем сохраняет их в `data/storage`. Размер чанков берётся из настроек (по умолчанию 1100 символов, overlap 180); метаданные дополняются названием, категорией, предполагаемым разделом и страницей.

## Запуск

### FastAPI

```bash
source .venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Эндпоинты:
- `GET /documents` — получить список документов в KB.
- `POST /documents` (alias `POST /documents/add`) — добавить файлы (`.docx`/`.pdf`) и инициировать ingest.
- `POST /documents/delete` — удалить документы из KB и переиндексировать (или очистить индекс, если документов не осталось).
- `POST /chat` — задать вопрос. В ответе прилетает текст `answer` + список `sources` (путь, документ, страница, раздел, категория, выдержка, score).

### Streamlit

```bash
source .venv/bin/activate
streamlit run ui/streamlit_app.py --server.port 8501 --server.address 0.0.0.0
```

- **Чат.** Вопросы к текущей базе документов с отображением источников.
- **Документы.** Загрузка и удаление `.docx`/`.pdf` c автоматической переиндексацией.
- **Настройки.** Позволяет менять промпт/пресеты, модель Groq, параметры генерации, `top_k`, стоп-последовательности, реранкер и флаг доменных правил.

## Тесты

```bash
source .venv/bin/activate
GROQ_API_KEY=dummy pytest
```

Покрытие включает нормализацию документов, создание чанков, работу ingest с кастомным каталогом и метаданными. Для проверки гибридного поиска добавляйте собственные тест-кейсы по необходимости.

## Практические советы

- Меняете chunk size или реранкер — пересоберите индекс (`python -m scripts.ingest`).
- При первом запуске cross-encoder скачивается с Hugging Face; если доступа нет, останется лёгкий keyword-реранкер (лог сообщает о fallback).
- Повреждённые PDF/DOCX при загрузке теперь пропускаются: в лог падает предупреждение, а индексация остальных файлов продолжается.
- Планируете эксплуатацию — подготовьте базовый набор документов, загрузите через UI, настройте промпт и прогоните типовые запросы в Streamlit до публикации.
